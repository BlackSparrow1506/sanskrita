# अभिकल्पनप्रतिमानानि — Design Patterns in संस्कृता

Design patterns are proven shapes for solving recurring programming problems. This guide shows the classic patterns in संस्कृता — and, more importantly, which ones the language makes *unnecessary*. Runnable code: `examples/प्रतिमानानि.सं`.

## The संस्कृता philosophy on patterns

Many famous patterns from the Java era are workarounds for missing language features. Because संस्कृता has **first-class functions** (a विधि is a value you can store and pass), several patterns collapse into one-liners. We teach the pattern *ideas* but use the simplest form the language allows.

## १. निर्माता (Factory)

**Problem:** code needs to create different kinds of objects depending on data.
**Shape:** one विधि receives a description and returns the right वर्गः instance.

```
विधि आकृतिनिर्माता(रूपम्, मापः) {
    यदि (रूपम् == "वृत्तम्") { फलम् सृज वृत्तम्(मापः)। }
    फलम् सृज चतुरस्रम्(मापः)।
}
```

Callers never write `सृज` themselves — they ask the factory. Adding a new shape changes one place only.

## २. दर्शकः (Observer)

**Problem:** when something happens, many parts of the program must react — without the announcer knowing who they are.
**Shape:** the घोषकः keeps a सूची of listeners and calls each on every announcement. In संस्कृता, listeners are simply **functions**:

```
विधि घोषय(वार्ता) {
    प्रत्येकम् श्रोता इति अयम्.श्रोतारः { श्रोता(वार्ता)। }
}
```

In Java this requires interfaces and anonymous classes. Here: योजय a विधि to a सूची.

## ३. उपायः (Strategy) — mostly unnecessary!

**Problem:** the same operation should follow different policies (normal price vs festival price).
**Java shape:** a Strategy interface + one class per policy + injection.
**संस्कृता shape:** pass the policy *as a function argument*:

```
विधि देयम्(मूल्यम्, नीतिः) { फलम् नीतिः(मूल्यम्)। }
देयम्(५००, उत्सवमूल्यम्)।
```

The entire pattern is one parameter. This is the "language features eliminate patterns" principle.

## ४. एकाकी (Singleton) — also mostly unnecessary!

**Problem:** exactly one shared instance (configuration, registry).
**Java shape:** private constructors, static getInstance(), thread locks.
**संस्कृता shape:** a top-level ध्रुव:

```
ध्रुव विन्यासः = {"भाषा": "संस्कृता"}।
```

ध्रुव guarantees the binding never changes; the file's top level guarantees there is exactly one. Done.

## Patterns that await future features

- **Iterator** — built in already: प्रत्येकम् … इति *is* the iterator pattern.
- **Decorator, Adapter** — practical once lambdas (anonymous विधयः) arrive.
- **Template Method** — needs class inheritance (planned).

## The one-sentence summary

Learn patterns as *vocabulary for design conversations* — but in संस्कृता, always ask first: "does a विधि-as-value or a ध्रुव already solve this?" Usually, yes.
